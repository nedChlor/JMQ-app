1. ОБЩИЕ ПОЛОЖЕНИЯ
================================================================================

1.1. Наименование проекта
    Мобильное приложение "JMQ Service Manual" (рабочее название).

1.2. Назначение
    Приложение предназначено для автомехаников сервисного центра. Оно предостав-
    ляет быстрый доступ к техническим инструкциям, сервисным руководствам и диа-
    гностическим кодам неисправностей (DTC) по обслуживаемым автомобилям.

1.3. Цель разработки
    Разработка кросс-платформенного мобильного приложения для оперативного
    поиска и просмотра технической документации по автомобилям, с возможностью
    диагностики по коду ошибки.

1.4. Целевая аудитория
    - Автомеханики сервисного центра
    - Технические специалисты по диагностике
    - Мастера-приёмщики (для быстрой сверки кодов неисправностей)

2. ТРЕБОВАНИЯ К ФУНКЦИОНАЛУ
================================================================================

2.1. Главный экран (Dashboard)
    2.1.1. Поисковая строка (полнотекстовый поиск) в верхней части экрана.
    2.1.2. Быстрый доступ к последним просмотренным инструкциям.
    2.1.3. Плитки категорий инструкций (Двигатель, Трансмиссия, Шасси, Кузов,
           Электрические системы, Калибровка).
    2.1.4. Поле быстрого ввода DTC-кода с кнопкой "Найти".
    2.1.5. Индикация выбранной модели автомобиля (с возможностью переключения
           при добавлении новых моделей в будущем).

2.2. Категории и навигация по инструкциям
    2.2.1. Иерархическая структура категорий (два уровня).
    2.2.2. Первый уровень: Двигатель, Трансмиссия, Шасси, Кузов,
           Электрические системы, Калибровка.
    2.2.3. Второй уровень (подкатегории):
           - Двигатель → HFC4GB2.4D (1.5T), HFC4GB2.4E (1.5T Euro6)
           - Трансмиссия → CVT Punch (вариатор), MF622D35 (мех. КПП)
           - Кузов → Структура кузова, Электрооборудование
           - Электрические системы → Принципиальные схемы, AVM (камера 360°),
             TPMS (давление в шинах), MP5
    2.2.4. Отображение количества файлов в каждой категории.
    2.2.5. Возможность поиска внутри выбранной категории.

2.3. Просмотр инструкций
    2.3.1. Отображение извлечённого текста документа для быстрого чтения.
    2.3.2. Встроенный PDF-вьювер для полноценного просмотра PDF-документов
           с поддержкой масштабирования, навигации по страницам.
    2.3.3. Кнопка "Открыть оригинал" — открытие PDF-файла встроенным
           просмотрщиком.
    2.3.4. Для файлов DOCX/PPTX — отображение извлечённого текста с пометкой
           об исходном формате.
    2.3.5. Сохранение истории просмотров.

2.4. Поиск по DTC-кодам
    2.4.1. Поле ввода кода неисправности (P, B, C, U коды).
    2.4.2. Автодополнение при вводе (подсказки по ранее найденным кодам).
    2.4.3. Результаты поиска:
           - DTC-код
           - Описание на русском языке (Chinese translation)
           - Описание на английском языке (DTC Meaning)
           - Контроллер (ECU), к которому относится код
           - Система автомобиля (ABS, EMS, BCM, SRS и т.д.)
    2.4.4. Фильтрация по системе/контроллеру (EMS, BCM, SRS, ESP и т.д.).
    2.4.5. Возможность поиска как по коду, так и по тексту описания.

2.5. Полнотекстовый поиск
    2.5.1. Поиск по всему тексту инструкций (FTS — Full-Text Search).
    2.5.2. Поиск по DTC-кодам и их описаниям.
    2.5.3. Подсветка найденных совпадений в результатах.
    2.5.4. Сортировка результатов по релевантности.
    2.5.5. Фильтр по категории инструкции.

2.6. Управление данными
    2.6.1. Автоматический импорт данных при первом запуске (из assets).
    2.6.2. Индикация процесса инициализации (прогресс-бар).
    2.6.3. Возможность добавления новых моделей автомобилей через обновление
           приложения (через магазин приложений).
    2.6.4. В перспективе: возможность загрузки дополнительных инструкций
           через Wi-Fi/LAN из внутренней сети сервисного центра.

5. МОДЕЛЬ ДАННЫХ
================================================================================

5.1. Основные сущности:

    5.1.1. Vehicle (Автомобиль)
        - id (INTEGER PK)
        - model_code (TEXT) — короткий код модели (J7, S3, S5...)
        - model_name_ru (TEXT) — название на русском (J7 / JS4)
        - model_name_en (TEXT) — название на английском
        - year (TEXT) — год выпуска

    5.1.2. Category (Категория инструкций)
        - id (INTEGER PK)
        - name_ru (TEXT)
        - parent_id (INTEGER FK → Category.id, NULL для корневых)
        - sort_order (INTEGER)
        - icon (TEXT) — название иконки

    5.1.3. Document (Документ-инструкция)
        - id (INTEGER PK)
        - vehicle_id (INTEGER FK → Vehicle.id)
        - category_id (INTEGER FK → Category.id)
        - title_ru (TEXT) — название на русском
        - original_filename (TEXT) — исходное имя файла
        - file_path (TEXT) — путь к файлу в assets
        - file_type (TEXT) — расширение (pdf, docx, pptx)
        - content_text (TEXT) — извлечённый текст для FTS
        - file_size (INTEGER) — размер файла в байтах
        - created_at (TEXT)

    5.1.4. DtcCode (Диагностический код неисправности)
        - id (INTEGER PK)
        - vehicle_id (INTEGER FK → Vehicle.id)
        - ecu (TEXT) — контроллер (EMS, BCM, SRS...)
        - code (TEXT) — DTC-код (P000A00, B100117...)
        - meaning_en (TEXT) — описание на английском
        - meaning_ru (TEXT) — описание на русском
        - description (TEXT) — подробное описание / условия возникновения

5.2. FTS (Full-Text Search) таблицы
    5.2.1. documents_fts — виртуальная таблица FTS5 для полнотекстового поиска
           по извлечённому тексту инструкций.
    5.2.2. dtc_fts — виртуальная таблица FTS5 для поиска по DTC-кодам
           и описаниям.

6. ПОДГОТОВКА ДАННЫХ
================================================================================

6.1. Исходные данные (J7/)
    6.1.1. 21 файл инструкций общим объёмом ~317 МБ.
    6.1.2. Форматы: PDF (5), DOCX (9), PPTX (7).
    6.1.3. Файлы структурированы по папкам-категориям.

6.2. Предобработка (Python-скрипт)
    6.2.1. Извлечение текста из DOCX — через библиотеку python-docx.
    6.2.2. Извлечение текста из PPTX — через библиотеку python-pptx.
    6.2.3. Извлечение текста из PDF — через PyMuPDF (fitz) или pdfplumber.
    6.2.4. Чтение DTC-таблиц из Excel — через openpyxl.
    6.2.5. Генерация JSON-файлов для импорта в SQLite на мобильном устройстве.

6.3. Импорт в приложение
    6.3.1. При первом запуске приложение копирует предобработанные JSON
           и PDF-файлы из assets в рабочую директорию.
    6.3.2. Создание и наполнение таблиц SQLite.
    6.3.3. Создание FTS-индексов.
    6.3.4. Отображение прогресса инициализации пользователю.

================================================================================

8.6. Приложение должно работать полностью офлайн (без доступа к интернету).

================================================================================
11. ГЛОССАРИЙ
================================================================================

DTC (Diagnostic Trouble Code) — диагностический код неисправности.
ECU (Electronic Control Unit) — электронный блок управления.
FTS (Full-Text Search) — полнотекстовый поиск.
AVM (Around View Monitor) — система кругового обзора 360°.
TPMS (Tire Pressure Monitoring System) — система контроля давления в шинах.
EMS (Engine Management System) — система управления двигателем.
BCM (Body Control Module) — блок управления кузовными системами.
SRS (Supplemental Restraint System) — система пассивной безопасности (подушки).
ABS (Anti-lock Braking System) — антиблокировочная система тормозов.
ESP (Electronic Stability Program) — система курсовой устойчивости.
TCU (Transmission Control Unit) — блок управления трансмиссией.
